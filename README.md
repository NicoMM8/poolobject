## 1. Objetivos de la Práctica

El objetivo principal de esta práctica es aprender a extraer métricas tanto del código que escribimos (el producto) como de nuestra forma de trabajar (el proceso). 

Para lograrlo, vamos a desarrollar de forma colaborativa una batería de pruebas para *Pool Object*, analizando cómo va evolucionando la cobertura del código gracias al uso de un entorno de Integración Continua (CI).


## 2. Descripción del Caso de Estudio (Enunciado)

Se simula un pequeño desarrollo de un producto software para realizar mediciones sobre él. El objetivo es establecer un caso de estudio dummy que sirva para caracterizar y evaluar tanto el producto desarrollado como el proceso colaborativo seguido. En concreto, el producto software es un código de test y el proceso es el análisis de la cobertura de pruebas en el tiempo.

**Descripción específica:**
Dado un código de ejemplo del patrón de diseño creacional *Pool Object*, se debe crear una batería de pruebas tal que las coberturas de sus clases sean del 100%. La batería de pruebas JUnit está contenida en la clase `ReuseblePoolTest.java`.
